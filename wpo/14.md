#### Use lazy loading for images

```
<img
  src={data}
  alt="Preview"
  className="results-image-preview"
  loading="lazy"
/>
```

Very often, webpages contain many images that contribute to data-usage and how fast a page can load. 
Most of those images are **off-screen (non-critical)**, requiring a user interaction, 
like scrolling, in order to view them. Similarly, many iframes, videos, and audio may be offscreen initially.

The loading attribute on an `<img>`, `<iframe>`, `<video>`, or `<audio>` element can be used to instruct the browser to defer loading of linked resources 
when elements are off-screen until the user scrolls near them. 
This allows non-critical resources to load only if needed, potentially speeding up initial page loads and reducing network usage.

#### Add width and height attribute to images

- Always set `width` and `height` on images (especially with lazy loading) to prevent layout shifts and improve Core Web Vitals.
- This is a best practice for both React and plain HTML.
