
## Core Web Vitals is an umbrella term for new quality factors of the resource

### LCP — Largest Contentful Paint
Determines how quickly the largest and the most important content on the site loads.

Reports the render time of the largest image or text block visible within the viewport relative to when the page first started loading.

Sites should strive to have the Largest Contentful Paint of 2.5 seconds or less.

### INP — Interaction to Next Paint (Replaces FID — First Input Delay)
Determines how quickly a site responds to different interactions from the user (click, scroll, etc.).

INP measures the time from when a user interacts with the page (i.e. when they click a link, tap on a button, or use a custom, JavaScript-powered control) to the time when the browser is actually able to present the result of the interaction.

Sites should strive to have an INP of 200 milliseconds or less.

### CLS — Cumulative Layout Shift
Determines how much the layout is shifted and if it's stable or not.

A layout shift occurs any time a visible element changes its position from one rendered frame to the next. CLS is a measure of the largest burst of layout shift scores for every unexpected layout shift that occurs during the entire lifespan of a page.

Sites should strive to have a CLS score of 0.1 or less.

### Таблица с основными метриками Core Web Vitals

| Метрика | Что измеряет | Целевое значение | Почему важно | Советы по улучшению |
| :-- | :-- | :-- | :-- | :-- |
| Largest Contentful Paint (LCP) | Время загрузки самого крупного видимого элемента контента (например, изображения, видео или блока текста). | ≤ 2.5 секунд | Чем быстрее пользователь видит основной контент, тем выше шанс положительного взаимодействия. | - Используйте сжатые изображения и форматы вроде WebP.<br> - Предзагружайте ключевые ресурсы (preload).<br> - Сократите время ответа сервера (улучшите TTFB). |
| First Input Delay (FID) | Время отклика браузера на первое взаимодействие пользователя с сайтом (например, клик, ввод текста или прокрутка). | ≤ 100 мс | Высокая задержка после действий пользователя создаёт негативный пользовательский опыт. | - Минимизируйте JavaScript и используйте async/defer.<br> - Оптимизируйте работу основного потока JS.<br> - Уберите ненужные блокирующие ресурсы. |
| Cumulative Layout Shift (CLS) | Уровень смещения элементов на странице (например, сдвиги кнопок, текстов, баннеров) во время её загрузки. | ≤ 0.1 | Резкие смещения контента раздражают пользователей и могут быть причиной ошибок (например, неправильное нажатие кнопки). | - Добавьте размеры для изображений и видео (width и height).<br> - Используйте статичные блоки для рекламы.<br> - Настройте fallback для шрифтов и минимизируйте задержки их загрузки. |


### Дополнительная информация обо всех метриках
| Показатель | Ключевая задача | Проблемы, если значение плохое |
| :-- | :-- | :-- |
| LCP | Загружать основной контент быстрее, чтобы пользователь сразу видел ключевой функционал. | Медленные сайты вызывают фрустрацию у пользователей, увеличивается показатель отказов (bounce rate). |
| FID | Быстро реагировать на действия пользователя после загрузки (убрать "подвисание" интерфейса). | Неспособность взаимодействовать с интерфейсом — пользователи могут покинуть сайт. |
| CLS | Поддерживать стабильный макет, чтобы элементы не прыгали во время загрузки. | Смещения элементов могут испортить навигацию, вызвать неверное нажатие или раздражение пользователей. |

### Минимальные и критические значения для каждой из метрик

| Метрика | Хорошо | Улучшить | Плохо |
| :-- | :-- | :-- | :-- |
| LCP | ≤ 2.5 секунд | 2.5–4 секунд | > 4 секунд |
| FID | ≤ 100 мс | 100–300 мс | > 300 мс |
| CLS | ≤ 0.1 | 0.1–0.25 | > 0.25 |
