### Step-by-Step: What does `new MyClass()` do?
#### 1. Creates a new empty object
- JavaScript creates a new, empty object in memory.
- This object will become your instance.
#### 2. Sets the new object's internal prototype
- The new object's internal prototype (`__proto__`) is set to point to `MyClass.prototype`.
- This means: If you access a property or method on the instance and it’s not found directly on the instance, JavaScript will look for it on `MyClass.prototype`.
#### 3. Executes the constructor function
- The constructor function inside your class is called, with this set to the new object.
- Any properties assigned to this inside the constructor become properties of the new instance.
#### 4. Returns the new object
- If the constructor doesn’t explicitly return an object, the new instance is returned.

### When is `MyClass.prototype` created?
- `MyClass.prototype` is created **automatically** by JavaScript when you **define the class**.
- This happens as soon as the **JavaScript engine reads your class declaration** or expression, even before you create any instances with new.

### `__proto__` vs `prototype`

| Property | Found on | Points to | Used for |
| :-- | :-- | :-- | :-- |
| prototype | Constructor | Prototype object for instances | Defining shared methods |
| __proto__ | Instance/object | The object's actual prototype | Inheritance lookup |

`__proto__` is used internally by JavaScript to look up properties and methods in the prototype chain.
`prototype` is a **property of constructor functions** (including classes). It defines the object that will become the prototype of all instances created by that constructor.
