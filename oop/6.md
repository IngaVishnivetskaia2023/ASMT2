## Inheritance and Super

```
class ParentClass {
  greet() {
    console.log("Hello from ParentClass");
  }
}

class NewClass extends ParentClass {
  greet() {
    console.log("Hello from NewClass");
    super.greet(); // Calls the method in ParentClass
  }
}

const instance = new NewClass();
instance.greet();
// Output:
// "Hello from NewClass"
// "Hello from ParentClass"
```

### Technical Details of JavaScript OOP
#### a. Prototype Chain
- In JavaScript, classes are syntactic sugar over the **prototype-based inheritance system**.
- When you create a class (`NewClass`) that extends another class (`ParentClass`), JavaScript sets up the prototype chain:
- `NewClass.prototype` inherits from `ParentClass.prototype`.
- Instances of `NewClass` have access to methods defined in both `NewClass.prototype` and `ParentClass.prototype`.
#### b. Method Resolution
- When a method is called on an object, JavaScript uses the prototype chain to resolve the method:
- It first looks for the method in the object itself (or its prototype).
- If the method is not found, it looks up the prototype chain.
- This continues until the method is found or the end of the prototype chain is reached (`null`).
#### c. super Keyword
- The `super` keyword is used to access **methods** or **constructors** of the **parent class**.
- In methods, `super.methodName()` calls the method in the parent class.
- In constructors, `super()` calls the parent class constructor.

❗ If the child class (NewClass) has its **own constructor**, it **MUST** call the parent class constructor using `super()` before accessing `this`.

```
class ParentClass {
  constructor(name) {
    this.name = name;
  }
}

class NewClass extends ParentClass {
  constructor(name, age) {
    super(name); // Calls ParentClass constructor
    this.age = age;
  }
}

const instance = new NewClass("Alice", 25);
console.log(instance.name); // Output: "Alice"
console.log(instance.age);  // Output: 25
```

## Class own properties

In JavaScript, class own properties refer to properties that **are directly defined on the instance of a class**, rather than being inherited from the class's prototype or shared across all instances. These properties are created when the class is instantiated and are unique to each instance of the class.

```
class Base {
  baseField = 10; // Own property, created on the instance
}

const baseInstance = new Base();
console.log(baseInstance.baseField); // 10
console.log(Base.prototype.baseField); // undefined (not on the prototype)
```

```
class Base {
  baseField = 10; // Own property, created on the instance
}

class Extended extends Base {
  extendedField = super.baseField; // undefined, not part of Base.prototype
}
```

- When `super.baseField` is used in `Extended`, JavaScript looks for `baseField` on `Base.prototype`, because `super` refers to the parent class's **prototype**.
- Since `baseField` is an **own property** of the **instance** and not defined on `Base.prototype`, `super.baseField` evaluates to `undefined`.


## Static Properties

Static properties are useful when you need to store data or functionality that is shared across all instances of a class or is independent of any specific instance.
<img width="764" height="427" alt="Screenshot (2240)" src="https://github.com/user-attachments/assets/f1e611b9-b6b8-49ea-a590-6af3c11c7db1" />

```
class Config {
  static API_URL = "https://api.example.com";
  static TIMEOUT = 5000; // Timeout in milliseconds
}

console.log(Config.API_URL); // "https://api.example.com"
console.log(Config.TIMEOUT); // 5000
```


```
class MathUtils {
  static PI = 3.14159;

  static calculateCircleArea(radius) {
    return this.PI * radius * radius;
  }
}

console.log(MathUtils.PI); // 3.14159
console.log(MathUtils.calculateCircleArea(5)); // 78.53975
```

## Newer OOP Features in JavaScript

### Getters and Setters

The main purpose of getters and setters is:
- They look like **SIMPLE PROPERTY** access (`obj.value`) or assignment (`obj.value = 10`)
- But **BEHIND the scenes**, they run functions that can:
- Calculate or process the value when you get it (getter)
- Validate, transform, or trigger actions when you set it (setter)

❗Getters in a class let you define a property that **looks like a regular value**, but is **actually calculated or processed when accessed**.

<img width="818" height="258" alt="Screenshot (2242)" src="https://github.com/user-attachments/assets/8974d5d0-12ce-40e9-beff-773514393ea7" />

### Getter Benefits

- Cleaner syntax: Access like a property (`obj.value`), not a method (`obj.getValue()`).
- Encapsulation: Hide internal logic or calculations from the user.
- Automatic calculation: Value is **always up-to-date** when accessed.
- Read-only: You can prevent direct modification (unless you add a setter).
