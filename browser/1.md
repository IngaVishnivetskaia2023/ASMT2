Modern browsers are complex, multi-process applications. 
Here's a simplified view of how the browser is structured and how browser-provided APIs fit into the architecture:

## Key Components of a Browser
### JavaScript Engine (e.g., V8):
- Executes JavaScript code.
- Manages the event loop and callback queue.
- Delegates tasks (e.g., network requests, timers) to other browser processes.

### Rendering Engine:
- Parses HTML and CSS to build the DOM and CSSOM.
- Combines the DOM and CSSOM to create the Render Tree.
- Handles layout, painting, and compositing to display the page.

### Networking Layer:
- Handles HTTP/HTTPS requests and responses.
- Manages connections to servers using protocols like TCP/IP.

### Storage Layer:
- Provides access to browser storage mechanisms like localStorage, sessionStorage, IndexedDB, and cookies.

### Browser Main Process:
- Coordinates communication between all the browser's components (e.g., rendering engine, networking layer, JavaScript engine).
- Manages tabs, windows, and user interactions.
