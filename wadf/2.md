## CSS methodologies

The most popular ones:

- Atomic CSS - One Rule for One Styling
-BEM - component-based approach, any component level class is a block, any its part is an element, the state of both block or element could be described with modifier
- ITCSS - Inverted Triangle CSS divides CSS structure into sections going up-down from more general to more specific
- SMACSS - separate our classes into groups Base rules, Layout rules, Modules, State rules, Theme rules, name and structure them accordingly
- OOCSS - introduces wide code reuse with extracting repeating CSS patterns as default classes
They can be used in combination together as well as independently. Take the best of each approach.

Методологии CSS помогают организовать и структурировать стили, облегчая их поддержку, масштабирование и понимание. Ниже представлен обзор самых популярных методологий с их ключевыми аспектами и рекомендациями.

| Методология | Описание | Принципы и особенности | Когда использовать |
| :-- | :-- | :-- | :-- |
| Atomic CSS | "Одна правило — один стиль". Каждое свойство (например, margin, padding, color) — это отдельный класс. | - Простой подход к переиспользованию.<br>- Классы несут в себе единичные стили (например, .text-red, .mt-5).<br>- Высокая детализация. | - Для проектов, где важны производительность и минимизация кода.<br>- Прекрасно работает для построения дизайн-систем или utility-first подходов. |
| BEM (Block-Element-Modifier) | Компонентный подход. Каждый компонент страницы — это блок. Составные части блока — это элементы. Модификаторы описывают состояние. | - Привязка к HTML-структуре:<br>  - block: .button;<br>  - block__element: .button__icon;<br>  - block--modifier: .button--active.<br>- Повышает читабельность и предсказуемость кода. | - Для больших команд, которым нужна строгая структура именования.<br>- Для проектов с многократным использованием компонентов. |
| ITCSS (Inverted Triangle CSS) | Иерархическая структура CSS в виде "перевернутого треугольника". Стили упорядочены от глобальных к локальным. | - Уровни: <br> 1️⃣ Settings (переменные)<br> 2️⃣ Tools (Mixin и функции)<br> 3️⃣ Generic (нормализация стилей)<br> 4️⃣ Elements (прямое форматирование HTML-элементов)<br> 5️⃣ Objects (макетные компоненты)<br> 6️⃣ Components (основные компоненты)<br> 7️⃣ Utilities (одноразовые классы, например hidden). | - Для масштабируемых проектов, где требуются чёткие правила организации классов.<br>- Подходит для проектов с долгосрочной поддержкой. |
| SMACSS (Scalable and Modular Architecture for CSS) | Делит CSS-классы на 5 групп: Base, Layout, Module, State, Theme. Структура определяет назначение классов. | - Base: Основные (HTML-элементы).<br> - Layout: Крупные секции сайта (шаблоны).<br> - Module: Повторяющиеся компоненты (например, кнопки).<br> - State: Отражает состояние компонентов (is-active, is-hidden).<br> - Theme: Тематизация (например, темные и светлые темы). | - Для подачи иерархии и разделения ответственности в проекте.<br>- Подходит для проектов с многими повторяющимися блоками и компонентами. |
| OOCSS (Object-Oriented CSS) | Объектно-ориентированный подход. Общие элементы разделяются на самостоятельные классы, которые переиспользуются. | - Два ключевых принципа: отделение внешнего вида от структуры.<br>- Разделение контейнеров и контента.<br>- Минимизирует дублирование кода. | - Для проектов с большим количеством повторяющихся стилей, где важна гибкость и переиспользование. |
